# YAGNI / KISS 数据管线改造计划

## 1. 核心原则

本仓库的职责应收敛为三件事：

1. 收集数据。
2. 清洗成统一格式。
3. 计算客观指标。

按照 YAGNI 和 KISS 原则，仓库不负责输出投资结论，不生成颜色状态，不生成买卖动作建议，不写主观方向解读。需要语义理解、新闻判断、交易决策的部分，应留给后续人工或联网分析环节处理。

## 2. 推荐接入数据

### 2.1 QQQ 价格数据

优先接入 QQQ 日线价格，NDX 或纳斯达克 100 指数可作为交叉验证数据。

建议数据源：

| 来源 | 用途 |
| --- | --- |
| Tiingo | 主价格源 |
| Alpha Vantage | 备用价格源 |
| yfinance | 本地回退，不作为 GitHub Actions 主接口 |

仓库只计算客观指标：

| 指标 | 说明 |
| --- | --- |
| 最新收盘价 | 记录最新日线收盘价 |
| 20 日收益率 | 基于历史收盘价计算 |
| 60 日收益率 | 基于历史收盘价计算 |
| 20 日年化波动率 | 基于收益率序列计算 |
| 当前回撤 | 基于历史高点计算 |
| 最大回撤 | 基于历史区间计算 |
| 50 日均线 | 基于收盘价计算 |
| 200 日均线 | 基于收盘价计算 |

### 2.2 FRED 宏观数据

优先接入 FRED 宏观序列，这类数据更新频率低、接口稳定，适合自动拉取并落表。

建议指标：

| 指标 | 建议来源 |
| --- | --- |
| 美国 10 年期国债收益率 | FRED |
| 美国 2 年期国债收益率 | FRED |
| 美国 3 个月国债收益率 | FRED |
| 有效联邦基金利率 | FRED |
| 消费者价格指数 | FRED |
| 个人消费支出价格指数 | FRED |
| 失业率 | FRED |

仓库只计算客观指标：

| 指标 | 说明 |
| --- | --- |
| 10 年期收益率当前值 | 只记录数值 |
| 2 年 / 10 年利差 | 2 年期收益率减 10 年期收益率 |
| 10 年期收益率近 1 个月变化 | 当前值减前值 |
| 通胀同比或环比 | 只计算数值 |
| 失业率最新值 | 只记录数值 |
| 数据发布日期 | 用于判断数据是否过期 |

### 2.3 QQQ 持仓数据

建议接入 QQQ 官方持仓数据。QQQ 持仓可用于计算权重集中度、行业暴露和头部持仓覆盖情况。

仓库应保存字段：

| 字段 | 说明 |
| --- | --- |
| 日期 | 持仓日期 |
| 股票代码 | 如 NVDA、AAPL |
| 公司名称 | 公司名 |
| 权重 | 占 QQQ 比例 |
| 行业 | 如信息技术、通信服务 |
| 数据来源 | Invesco 或备用来源 |

仓库可以计算客观指标：

| 指标 | 说明 |
| --- | --- |
| 前 5 大持仓权重合计 | 客观集中度 |
| 前 10 大持仓权重合计 | 客观集中度 |
| 单一最大持仓权重 | 客观集中度 |
| 行业权重合计 | 客观行业暴露 |

### 2.4 市场广度数据

市场广度建议由持仓或成分股价格自行计算，不优先抓取网页指标。这样可以减少网页结构变化、权限套餐和额外接口带来的维护成本。

建议实现方式：

1. 使用 QQQ 持仓或纳斯达克 100 成分股列表。
2. 拉取每只股票的日线价格。
3. 基于统一价格数据计算广度指标。

仓库可以计算：

| 指标 | 计算方式 |
| --- | --- |
| 上涨家数 | 今日收盘价 > 昨日收盘价 |
| 下跌家数 | 今日收盘价 < 昨日收盘价 |
| 上涨家数占比 | 上涨家数 / 可计算股票数 |
| 高于 20 日均线比例 | 当前价 > 20 日均线的股票数 / 可计算股票数 |
| 高于 50 日均线比例 | 当前价 > 50 日均线的股票数 / 可计算股票数 |
| 高于 200 日均线比例 | 当前价 > 200 日均线的股票数 / 可计算股票数 |
| 20 日新高数量 | 当前价 >= 过去 20 日最高价 |
| 20 日新低数量 | 当前价 <= 过去 20 日最低价 |

### 2.5 前 10 / 前 20 大持仓报价与轻量基本面

建议只做轻量版，优先覆盖 QQQ 前 10 或前 20 大持仓。FMP 可作为报价和轻量基本面来源，但不应在第一阶段扩展到完整财报三表。

建议字段：

| 字段 | 是否建议 |
| --- | --- |
| 当前价 | 建议 |
| 涨跌幅 | 建议 |
| 市值 | 建议 |
| 市盈率 | 可选 |
| 每股收益 | 可选 |
| 下次财报日期 | 可选 |
| 分析师目标价 | 暂缓 |
| 完整利润表 | 暂缓 |
| 资产负债表 | 暂缓 |
| 现金流量表 | 暂缓 |

仓库可计算数据质量指标：

| 指标 | 说明 |
| --- | --- |
| 前 10 大持仓报价成功率 | 数据质量指标 |
| 前 10 大持仓权重覆盖率 | 数据覆盖指标 |
| 缺失股票列表 | 方便排查 |
| 接口错误原因 | 方便维护 |

## 3. 暂缓或不建议自动接入的数据

| 数据 | 原因 | 建议处理方式 |
| --- | --- | --- |
| 新闻情绪 | 需要语义理解，容易误判 | 分析时联网处理 |
| 当日盘中实时价格 | 与日线系统不一致，容易制造噪音 | 分析时临时补充 |
| 美联储讲话全文 | 解析复杂，不适合仓库自动处理 | 分析时联网读取重点新闻 |
| 期权隐含波动率 | 接口复杂，当前不是必需 | 后续需要再加 |
| 资金流入流出 | 数据源权限复杂 | 暂缓 |
| 分析师评级 | 容易形成主观判断 | 暂缓 |
| 完整财报三表 | 数据量大，短期用不上 | 后续单独扩展 |
| 复杂网页抓取 | 网页结构和权限不稳定 | 优先用 API 或已有价格数据自行计算 |

## 4. 推荐输出结构

仓库只输出数据文件，不输出投资结论。

```text
reports/latest/
  manifest.json
  price_daily.csv
  price_metrics.csv
  macro_daily.csv
  macro_metrics.csv
  qqq_holdings.csv
  breadth_metrics.csv
  top_holdings_quotes.csv
  data_quality.csv
  run_log.csv
```

| 文件 | 内容 |
| --- | --- |
| manifest.json | 本次运行日期、生成时间、文件清单 |
| price_daily.csv | QQQ、可选 NDX 的原始日线数据 |
| price_metrics.csv | 收益率、波动率、回撤、均线 |
| macro_daily.csv | FRED 宏观原始数据 |
| macro_metrics.csv | 利差、变化值、发布日期 |
| qqq_holdings.csv | QQQ 持仓 |
| breadth_metrics.csv | 市场广度计算结果 |
| top_holdings_quotes.csv | 前 10 或前 20 持仓报价和轻量基本面 |
| data_quality.csv | 数据源是否成功、缺失字段、缺失股票 |
| run_log.csv | 运行日志 |

## 6. 分阶段路线

### 第一阶段：最小可行版本

第一阶段只做 4 个模块，先保证稳定、可审计、容易维护。

| 模块 | 数据源 | 输出 |
| --- | --- | --- |
| QQQ 价格 | Tiingo，Alpha Vantage 备用 | price_daily.csv、price_metrics.csv |
| 宏观 | FRED | macro_daily.csv、macro_metrics.csv |
| QQQ 持仓 | Invesco，失败则保留上一期并记录 | qqq_holdings.csv |
| 市场广度 | 用持仓股票价格自行计算 | breadth_metrics.csv |

第一阶段已经足够支撑后续分析读取仓库数据，再由分析环节单独给出判断。

### 第二阶段：稳定后扩展

| 模块 | 何时加 |
| --- | --- |
| 前 20 大持仓报价 | 第一阶段稳定后 |
| FMP 轻量基本面 | 报价稳定后 |
| 纳斯达克 100 完整成分股 | 如果发现 QQQ 持仓不能满足市场广度 |
| 新高 / 新低 | 市场广度稳定后 |
| 多数据源交叉验证 | 数据缺失或错误较多时再加 |

## 7. 最终目标

仓库最终职责应改造为：

> 每日收集 QQQ、宏观、持仓、市场广度、重点股票报价数据，并计算客观指标；不输出投资判断、不输出颜色状态、不输出买卖建议。

最合适优先接入的数据是：

1. QQQ 日线价格。
2. FRED 宏观数据。
3. QQQ 官方持仓。
4. 基于持仓股票价格计算的市场广度。
5. 前 10 或前 20 大持仓报价与轻量基本面。

暂时不要接入新闻情绪、期权、资金流、完整财报、分析师评级和复杂网页抓取。
