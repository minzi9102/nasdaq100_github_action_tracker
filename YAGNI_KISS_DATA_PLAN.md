# YAGNI / KISS 数据管线状态

本文记录原改造计划的当前落地状态。实际运行方式以代码、配置和 [README.md](README.md) 为准。

## 已落地

- 仓库职责收敛为数据采集、标准化、客观指标计算和质量记录。
- 不生成颜色状态、方向判断、买卖动作或投资建议。
- QQQ 价格使用 Alpha Vantage compact 更新统一缓存，Tiingo/Twelve Data 负责历史补齐。
- FRED 提供利率、通胀和就业序列。
- Invesco 官方 API 提供 QQQ 全量持仓，并派生股票持仓池。
- Twelve Data 提供前 20 大股票持仓报价。
- 市场广度基于合格历史缓存计算，并可叠加最新报价；目标日对齐通过独立字段审计。
- 缓存维护以 QQQ 最新有效收盘日为目标，只增量请求尚未对齐的标的。
- 输出 `breadth_constituents.csv`，解释每只成分股的日期、涨跌和 recent/strict 资格。
- 输出统一 `model_input_metrics.csv`、数据质量、API 用量、失败诊断和严格 manifest。
- 历史缓存维护、Twelve Data 修复和 provider 能力探测已拆分为独立工作流。

## 当前简化边界

- FMP 不参与生产报价或市场广度，只做能力探测。
- 不自动接入新闻情绪、分析师评级、期权、资金流或完整财报三表。
- 不依赖复杂网页抓取；优先使用官方 API 和本地可审计缓存。
- 日常市场广度不现场批量回填历史，避免生产日报被外部额度和限速拖垮。
- `model_input_metrics.csv` 只保存数值、日期、来源、覆盖率和质量说明。

## 数据质量门槛

历史缓存进入市场广度计算前必须：

- 至少包含 220 条有效日线记录。
- 最新记录距运行日期不超过 5 个自然日。

上述门槛只定义 `is_qualified`。分析目标日一致性时还必须检查 `is_target_date`、`needs_target_update` 和 `breadth_constituents.csv`；不能把“新鲜”直接等同于“已对齐目标交易日”。

分析前还应检查：

- 历史和报价覆盖率。
- 目标日期、对齐数量和缓存日期范围。
- 前 10、前 20 报价覆盖率。
- 高权重缺失标的。
- MA200 是否可用。
- provider 是否限速或回退。

## 后续改动原则

只有满足以下条件时才增加新数据源或指标：

1. 当前输出无法回答明确问题。
2. 数据源拥有稳定接口、清晰许可和可控额度。
3. 可以记录实际 provider、缺失、限速和回退。
4. 可以通过自动测试固定输出契约。
5. 不把主观分析重新塞回数据流水线。

新增 provider 的实际步骤见 [docs/ADD_PROVIDER.md](docs/ADD_PROVIDER.md)。
